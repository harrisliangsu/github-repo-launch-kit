# 引爆文骨架

发 HN / 即刻 / V2EX / 知乎 / dev.to 都能复用的结构。
**目标是「让没听说过你的人想点开 repo」，不是「介绍 repo 多么完整」。**

---

## 标题（决定 80% 的点击率）

写法：**具体痛点 + 反常识 / 反 hype 的钩子**。

❌ 我做了一个 LLM 微调的开源仓库
✅ 我用 MacBook 跑通了 SFT/LoRA/DPO 的 10 节课，公开给所有人

❌ 介绍 claude-update：一个 Claude Code 升级工具
✅ 每次 `claude update` 后我都搞不清自己升到了什么版本，所以做了 claude-update

> 标题里要有「我」「具体场景」「反 hype」三件之一。

---

## 第一段（决定 90% 的留存）

不要从 "I built X" 开始。从**用户的真实痛苦**开始。

```
模板：
  上下文：你是 <某种身份>，你想做 <具体的事>，但你卡在 <具体的痛点>。
  钩子：你尝试过 <主流方案 A/B/C>，但每次都 <为什么不够好>。
  转折：直到你发现 <你做的项目>。
```

---

## 第二段：踩了哪些坑

具体到代码 / 命令 / 参数 / 时间。这是文章里**最难复制、最值得读**的部分。

> 「我以为 LoRA rank 越大越好，跑了一晚上才发现 rank=8 和 rank=64 在我的数据上 loss 几乎一样，
> 但 rank=64 让 Apple Silicon 的统一内存压到了 27GB，机器风扇响了 4 小时。」

每个坑用 1-2 段，3-5 个坑就够。**带数字、带配置、带截图。**

---

## 第三段：所以这个项目长什么样

放 GIF / 截图 / 一段「30 秒能跑」的命令，不要列 feature 表。

```bash
git clone …
./bootstrap.sh
# 看到 X
```

---

## 第四段：这个项目不是什么

**反向定位**比正向定位更可信。

> 这不是 LLaMA-Factory 的替代品。LLaMA-Factory 适合用大显存追求效果，
> 这个仓库适合在本地一台 Mac 上把流程跑明白。

---

## 第五段：欢迎挑刺

具体的 ask，不要说「欢迎贡献」。

> 我特别想知道：
> - 在 4090 / 3090 / Apple M3 上有没有跑不通的地方
> - LoRA 这一节的解释是不是你能看懂
> - 你期待但仓库还没做的下一节课

---

## 平台微调

| 平台 | 调整点 |
|---|---|
| Hacker News | 标题用 `Show HN: <一句话>`，正文压到 ~300 词，第一条评论自己补充 backstory |
| Reddit | 不要带营销味，第一段就要有真材实料；遵守每个 sub 的自我推广规则（10:1） |
| 即刻 | 200-300 字，开头一句钩子，结尾问句邀请讨论 |
| V2EX | 节点选 `分享创造`，标题不要 emoji，正文务实 |
| 小红书 | 标题带数字 + 痛点，配图比文字重要，要有「过程感」截图 |
| 知乎 | 走「我在 X 时遇到 Y，所以做了 Z」长文路线，2000-3000 字 |
| dev.to / Medium | 英文，技术细节多放，配代码块 |
| X / Twitter | 拆成 thread，第一条带 GIF + repo 链接 |

---

## 时间窗口

- HN：周二 / 周三美西早 8-10 点（北京时间凌晨 12-2 点）
- Reddit：与 sub 的活跃时段对齐，r/MachineLearning 美东上午
- 即刻 / 小红书：晚 8-10 点
- X：早 9 点 / 晚 9 点（你时区）

---

## 发完之后 30 分钟内

- 盯住每条评论，**前 30 分钟的回复速度决定算法权重**
- 不要和钓鱼 / 喷子较真，第一条理性回复 + 不再回应
- 把高质量提问做成 GitHub Discussions，回复里贴链接
- 截图前 N 条评论，作为下一篇文章的素材
